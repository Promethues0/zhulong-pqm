> 版本 v1.0 · 面向：安全/密码工程师、CISO/CIO、DevSecOps、合规与运维

## 2. 核心理念：五阶段闭环

```
① 发现 ──▶ ② 建档(CBOM) ──▶ ③ 评估(五维) ──▶ ④ 改造(编排设备) ──▶ ⑤ 监测
   ▲                                                                      │
   └──────────────────── 复评回灌（漂移/情报触发重评分） ◀────────────────┘
```

- **① 发现**：对现网做 Agentless 探测，找出在用的密码算法（TLS/证书/SSH…）。
- **② 建档**：把发现结果归并成「密码使用点（CBOM）」清单，去重、留证据。
- **③ 评估**：用五维模型给每个使用点打分、定优先级（P1–P4）、标 HNDL。
- **④ 改造**：按剧本编排烛龙网关 / HSM / CA 下发后量子（混合）算法。
- **⑤ 监测**：持续复扫、SLO 守护、证书预警、威胁情报；发现回退或新风险自动回流评估。

## 3. 快速上手（10 分钟跑通一遍）

1. **发现**：进入「密码学发现」→ 新建扫描，目标填 `example.com:443`（每行一个），暴露面选「公网」→ 发起。几秒后任务「已完成」，弹出「新增 N 项密码使用点」。
2. **看清单**：点横幅「前往密码使用点清单」，可见刚入库的资产；点行展开详情（算法、五维分、证据链、命中规则）。
3. **评分**：进入「风险评估」，选一个资产，用五维下拉打分（或点预设画像一键填入），右侧实时出综合分与 P 级，保存。
4. **改造**：回「密码使用点清单」点某 P1 资产详情里的「发起改造」→ 跳「改造编排」并预填；选设备与剧本，「执行改造」，观察步骤时间线。
5. **验收**：改造完成后在「改造编排」点「一键验收」→「验收自动化」跑套件 → 生成报告 → 签署。
6. **监测**：进入「持续监测」看 SLO 卡、告警、证书到期；进入「摸底报告」一键生成整体态势报告。

## 4. 功能模块详解（按菜单）

### 4.1 进度仪表板
全局总览：密码使用点总数、P1 数、HNDL 数、极高风险数、平均风险分；资产分层分布（L1–L4）；优先级汇总（P1–P4 数量与均分）；最近扫描任务；**迁移进度趋势**（折线，点「采集今日快照」即时出点，多日成曲线）。

### 4.2 密码学发现（①）
- **新建扫描**：任务名 + 目标（每行一个）+ 暴露面（内网/DMZ/公网，影响 D5 暴露面评分）。Agentless，无需在目标装 Agent。
- **目标写法**：支持 `host`、`host:port`、`域名`，也可**直接粘贴完整网址**（`https://host/path` 会自动取主机名，默认 443）。
- **导入证据**：够不着目标时，可粘贴 **PEM 证书**、上传 **SBOM（CycloneDX/Syft）JSON**，或上传 **抓包（.pcap/.pcapng）** 直接建档（无需网络可达）。
- **后量子识别**：主动扫描与被动抓包都会解析 TLS 密钥交换组，识别混合/后量子协商（X25519MLKEM768、铜锁国密 curveSM2MLKEM768 等）并按「已迁移」建档评分——这是旧版看不到的信号。
- **任务列表**：状态自动轮询；点「结果」看每个端点的探测**状态**（成功/失败+原因）、TLS 版本、算法、证书有效期与**命中规则**。
- **探测失败会写明原因**（连接超时/被拒绝/无路由/域名解析失败/非 TLS 服务），不再静默出空结果。
- ⚠️ **关键**：平台只能扫到「**运行后端的那台机器**」网络可达的目标。扫公网 HTTPS 用本页；扫**内网/本地**资产用下面的「主机 Agent」或「导入证据」。

### 4.2.1 主机 Agent（补盲：内网 / 本地资产）
当目标在内网、仅监听 `127.0.0.1`、或被防火墙/安全组挡在外时，Agentless 网络扫描够不着。这时在**目标主机上本地运行 Agent**，把本机的密码学使用点采集后回报平台。

**采集范围（正是 Agentless 拿不到的）**：

| 采集项 | 说明 |
|---|---|
| 本机监听的 TLS 服务证书 | 含仅绑 `127.0.0.1` / 被安全组挡在外的端口 |
| 磁盘上的证书文件 | `/etc/ssl`、nginx、`/opt/*/etc/tls` 等，未对外提供的也能采 |
| **SSH 主机密钥**类型与位数 | 远程无法读取（弱 DSA/1024 等会被识别并按高风险评分） |
| 本机加密库版本（openssl→SBOM） | 远程不可能拿到 |

**Agent 已产品化为纯 Go 免 CGO 单二进制** `zhulong-pqm-agent`（信创鲲鹏/飞腾 amd64+arm64，交付包 `bin/` 内）。相比旧脚本，新增两大能力：

- **进程×加密库映射**（核心）：读 `/proc/<pid>/maps` 找出每个进程实际加载的密码库（OpenSSL/铜锁/openHiTLS/GmSSL/BoringSSL/AWS-LC/SymCrypt/liboqs/PQMagic…），并判定该库**是否具备后量子能力**——例如「nginx 加载 OpenSSL 3.5.0」判为后量子就绪（D1=15），「加载 OpenSSL 3.0.2」判经典（D1=70）。同 soname 歧义（`libcrypto.so.3` 可能是 OpenSSL 或铜锁）用版本串/符号消歧。
- **专属身份 + 归属**：Agent 用管理员注册时一次性下发的 **API Key** 上报（不再用 admin 密码），资产自动盖 `ReportedBy=<agentId>` 归属到具体主机。

**三步用法**：

```bash
# 1) 管理员在控制台注册一个 Agent，拿到一次性 apiKey（仅显示一次，务必保存）
curl -X POST http://<平台>/api/v1/agents -H "Authorization: Bearer <管理员token>" \
  -H 'Content-Type: application/json' \
  -d '{"hostname":"web-01","kind":"host","labels":["生产","DMZ"]}'
#   → {"agent":{"agentId":"agent-xxxx",...}, "apiKey":"zpqm-agent-...."}

# 2) 把 zhulong-pqm-agent 二进制拷到目标主机，用 apiKey 运行（只读，不改主机任何东西）
ZPQM_AGENT_SERVER=http://<平台> ZPQM_AGENT_KEY=zpqm-agent-.... \
  ./zhulong-pqm-agent --once

# 3) 到「密码使用点清单」按来源=agent 筛选，即见本机采集并已评分的使用点（含进程×库/证书/SSH主机密钥/监听服务）
```

采集范围见上表 + **进程×加密库**。常态化采集可加 `--interval 3600`（每小时）或放进 systemd timer。详细安装、参数、systemd 部署见 **《主机 Agent 安装手册》** `docs/主机Agent安装手册.md`。

> 旧 bash 脚本 `agent/zhulong-pqm-agent.sh` 仍保留可用（走用户 JWT 导入通道），但推荐用新二进制（进程×库发现 + 专属身份归属）。

### 4.3 规则库
内置 **30 条量子脆弱性检查规则**（L1 网络协议 / L2 协议传输 / L3 数据存储 / L4 硬件根信任，按 8/8/8/6 分布；P1 高优 14 条、极高 7 条）。可按层级/优先级/风险筛选；「覆盖度矩阵」展示各层 × 各发现方式（M1 主动 / M3 Agent / M4 SBOM / M5 证书 …）的覆盖情况。

### 4.4 密码使用点清单 · CBOM（②）
- 资产表：名称/系统/层级/算法/暴露面/综合分/分级/HNDL；支持层级·分级·HNDL·关键字筛选。
- **新增 / 编辑 / 删除**资产（状态流转受状态机约束，非法迁移会被拒）。
- **合并重复**：多来源（扫描+导入）同一对象按指纹聚簇，合并为单一使用点并保留各来源证据。
- 详情抽屉：五维分、建议迁移算法、证书到期、**证据链**（来源/规则/哈希固化）、**评分历史**时间线、「发起改造」按钮。
- 顶部「**导出 CBOM**」（CycloneDX JSON）、「导入 CBOM」。

### 4.5 CBOM 快照
把某一时刻的清单冻结为命名快照；选两个快照「对比」，展示新增/移除资产、**算法变更**（如 ECDH→X25519+ML-KEM-768 标「迁移进展」）、算法分布环比——用于量化迁移进度。

### 4.6 风险评估（③）
五维加权模型：**综合分 = D1×30% + D2×25% + D3×20% + D4×15% + D5×10%**
- D1 算法脆弱性 / D2 数据敏感度 / D3 数据生命周期 / D4 迁移复杂度 / D5 暴露面。
- 分级：**P1≥75（0–3 月）、P2 50–74（3–6 月）、P3 25–49（6–12 月）、P4<25（持续监控）**。
- **后量子双维建模**：每个使用点分「**密钥交换维**」与「**认证维**」两条独立迁移线。识别到混合/后量子密钥交换（X25519MLKEM768、铜锁 curveSM2MLKEM768 等）即判该维**已迁移**，D1 由 70/90（经典）降到 **15（混合）/10（纯 PQC）**。
- **HNDL 标记**：D2≥60 且 D3≥60（先存后解高危）标红；但**密钥交换维一旦迁到混合/后量子，HNDL 自动清除并显示「已缓解」**——因为「先抓后解」打的正是密钥交换。这把「迁了 KEX 的收益」量化了出来。
- 操作：选资产→五维下拉打分（实时出分/级/HNDL）→保存；提供 7 个**预设画像**（内部根CA / SSL VPN / 对外TLS / 长期档案 / IoT / 数据库 / 代码签名）一键填入；右侧优先级汇总看板。

### 4.7 风险登记（③）
- 风险台账：可按优先级/HNDL/层级筛选，P1 行高亮、HNDL 标红，含建议迁移算法；「**导出 CSV**」。
- **专家模式调权**：新建权重方案（五维之和须=100），「激活并复算」对全量资产按新权重重算并显示 P1–P4 迁移前后对比。默认锁标准权重 30/25/20/15/10。
- 「批量复算」：按当前生效方案重算全部资产。

### 4.8 改造编排（④）
改造主线 = **编排外部设备**（按「轨道 → 步骤 → 执行体 → 交付 → 验收」）。
- **改造工单**：新建（选资产或手填 + 选轨道剧本 + 选设备 + 目标算法）→「执行改造」→ 步骤时间线实时推进 → 证据/交付物 → 可「回滚」。离线设备的步骤会**诚实标注「模拟」**，不谎报成功。
- **设备纳管**：登记烛龙 IPSEC/SSL VPN 国密网关、HSM、CA；「连通性测试」探测在线状态与时延。
- **剧本库**：5 条内置轨道（外部TLS混合KEM、根CA/PKI混合根CA(HSM)、SSL VPN混合KEM(ke1_mlkem)、代码签名双签名、国密SM2+ML-KEM），每条含步骤、交付物、验收口径。

### 4.9 验收自动化（④）
- 把迁移验收做成可重复的 **47 项测试套件**（协议层 TLS 混合握手/IKEv2/OCSP/ML-DSA、降级攻击、混合密钥材料溯源、性能基准、兼容性矩阵）。
- 对改造工单「一键验收」→ 逐项判定（通过/有条件/未通过，模拟项诚实标注）→ **Gate 门禁**（通过≥44、有条件≤3、未通过=0、P1 全覆盖）→ 生成报告 → **签署流转**（DRAFT→评审→签署，签署后资产置「已验收」）。

### 4.10 持续监测（⑤）
- **SLO 八卡**：握手失败率(≤0.1%)、p99 延迟(≤46.2ms)、吞吐降幅(≤6.5%)、IKEv2 建立、CBOM 新鲜度、证书覆盖等，越阈红/临界橙/正常绿。
- **告警台账**：密码漂移（已迁混合回退到经典→P1）、证书到期、SLO 越界、情报触发；可确认/闭合/一键复评。
- **证书到期**（90 天内分级预警）、**复评队列**、**SLO 趋势图**、**遗留风险登记 R-001..**、**量子威胁情报**（录入/拉取，命中算法可批量触发复评回流评估）。

### 4.11 摸底报告
一键生成《后量子摸底报告》（Markdown，可作汇报）：总体态势、算法分布总览、按层级分布、P1 清单、HNDL 专项、算法选型建议（混合过渡→长期目标）、迁移批次建议、处置建议。历史报告可回看。

### 4.12 审计日志（仅 admin）
记录扫描/评分/改造/验收/设备/导出等敏感操作（操作人/时间/模块/动作/对象/结果/IP）；可筛选、**导出 CSV**。

### 4.13 系统设置
四个分页：**扫描默认**（暴露面/端口/超时/并发）、**SLO 阈值**（可改可一键恢复默认）、**评分权重**（只读展示，调权去「风险登记/专家模式」）、**威胁情报源 & 数据保存期**。

## 5. 角色与权限（RBAC）

| 角色 | 能力 |
|---|---|
| **admin** | 全部权限 + 用户管理 + 审计日志 + 系统设置 |
| **operator** | 读 + 所有写操作（扫描/评分/改造/验收/设备/导入导出） |
| **viewer** | 仅只读；任何写操作返回 403 并记审计 |

写操作统一受控；删除/降级最后一个 admin 会被拒。

## 6. 典型端到端工作流

> 「改一类资产」的最小闭环：

1. 发现 → 扫描对外 TLS / 导入证据；
2. 清单 → 合并去重，确认资产；
3. 评估 → 打分定级，P1/HNDL 优先；
4. 风险登记 → （可选）按行业调权复算，导出台账上报；
5. 改造编排 → 选剧本+设备，执行混合 KEM 下发；
6. 验收自动化 → 跑套件、过 Gate、签署；
7. 持续监测 → 纳管，复扫守护，漂移/情报自动回流评估；
8. 摸底报告 → 生成阶段性态势，向 CISO 汇报。

## 7. 常见问题（FAQ）

- **扫描没结果 / 域名解析失败 / 连接超时？** 平台只能扫到运行后端那台机可达的目标。公网 HTTPS 直接扫；**内网/本地资产改用「主机 Agent」（见 4.2.1）或「导入证据」**。目标可直接粘贴网址，失败时结果行会写明原因。
- **弹窗/下拉看不清？** 已修复（浮层实底）；如仍异常请强制刷新清缓存。
- **改造步骤显示「模拟」？** 表示对应设备当前离线/不可达，平台诚实标注而非谎报；设备上线后重跑即真实下发。
- **验收无法签署？** Gate 未达标（有未通过项，或有条件项未挂遗留风险）；按提示补齐。
- **趋势只有一个点？** 趋势按天累积，点「采集今日快照」先落一个点，多日后成曲线。
- **国密/PQC 算法口径？** 密钥封装 ML-KEM-768、签名 ML-DSA-65、国密融合 SM2+ML-KEM / SM2+ML-DSA（过渡），对齐 NIST FIPS 203/204/205。

## 8. 部署 / 运维速查（管理员）

- **云端服务**：后端 `systemctl status zhulong-pqm`（127.0.0.1:8099，systemd 自启）；日志 `journalctl -u zhulong-pqm -f`；前端经 nginx `/pqm/` 提供。
- **私有化安装**：见 `deploy/README.md`（`build.sh` 交叉编译 amd64/arm64 → `package.sh` 打包 → 目标机 `sudo ./install.sh`）。
- **数据库**：SQLite，云端在 `/var/lib/zhulong-pqm/zhulong-pqm.db`（备份直接拷该文件）。
- **更新前端**：本地 `vite build --base=/pqm/` → 覆盖 `/opt/zhulong-pqm/web`。
- **回滚下线**：`sudo systemctl disable --now zhulong-pqm`（不影响同机的烛龙控制台）。
